# Trademark Policy

English | [日本語](TRADEMARK.ja.md)

Policy for using the **zenchi-zenno** name, logos, and related marks.

**Related:** [GOVERNANCE.md](GOVERNANCE.md) · [docs/commercial-boundary.md](docs/commercial-boundary.md)

---

## Ownership

The **zenchi-zenno** name, wordmark, and any project logos are trademarks of the zenchi-zenno project / maintainers (the “Project”).

Software is licensed under the [LICENSE](LICENSE) (MIT as of Phase 0–1). **Trademark rights are not granted by the software license.** Using the code does not grant permission to use Project marks beyond this policy.

---

## Allowed without permission

You may:

1. **Refer to the project by name** factually (“compatible with zenchi-zenno”, “fork of zenchi-zenno”).
2. **Link to** the official repository and documentation.
3. **Use the name in documentation** about integrating with or extending zenchi-zenno.
4. **Indicate compatibility** accurately (“implements the zenchi-zenno Connector SPI”).

---

## Requires permission

You need written permission from Project maintainers to:

1. Use **zenchi-zenno** (or confusingly similar names) as the **product name** of a fork, hosted service, or commercial offering.
2. Use Project **logos** in marketing, app stores, or paid ads.
3. Imply **official endorsement**, partnership, or “official cloud” status.
4. Register domains, organizations, or social handles that suggest you are the Project (e.g. `zenchi-zenno-official`, `zenchi-zenno.cloud` implying official status).

---

## Forks

Forks are welcome under the software license. Forks **must**:

- Change the product/display name if distributed as a competing product or hosted service
- Not use Project logos
- Not claim to be the official zenchi-zenno distribution

Suggested renames: make clear affiliation (e.g. `acme-knowledge-os (zenchi-zenno fork)` in README) without presenting as upstream.

---

## Commercial and cloud offerings

Official managed offerings will be named distinctly when shipped (e.g. branding under Project control such as **zenchi-zenno Cloud**). Third-party hosted services using the OSS core must:

- Not use confusingly similar marks without permission
- Comply with [docs/commercial-boundary.md](docs/commercial-boundary.md) for describing what is OSS vs their SaaS add-ons

---

## Reporting misuse

Report suspected trademark misuse via a GitHub issue titled `[Trademark]` or by contacting maintainers listed in repository ownership settings.

---

## Updates

This policy may be updated as the Project grows. Material changes will be announced via Discussions and `CHANGELOG` / release notes when applicable.
