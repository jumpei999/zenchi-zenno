# セキュリティポリシー

zenchi-zenno はローカルファーストの知識オペレーティングシステムです。セキュリティ上の問題は、**個人の知識ストア**、コネクタの認証情報、取り込みパイプラインに影響しうるものです。脆弱性は責任ある方法で報告してください。

## サポート対象バージョン

| バージョン       | サポート                               |
| ---------------- | -------------------------------------- |
| `main`（最新）   | あり                                   |
| タグ付きリリース | より新しいリリースに置き換わるまであり |
| 古いタグ         | ベストエフォートのみ                   |

## 脆弱性の報告方法

セキュリティ脆弱性については、**公開の GitHub Issue を開かないでください**。

次のいずれかを使用してください:

1. **GitHub Private Vulnerability Reporting**（リポジトリ公開後は推奨）  
   リポジトリ → **Security** → **Report a vulnerability**

2. **メンテナーへの連絡**  
   非公開報告が利用できない場合は、[GOVERNANCE.md](GOVERNANCE.md) に記載のメンテナーへの非公開チャネルで連絡してください。公開 Issue や Discussions に exploit の詳細を投稿しないでください。

## 記載してほしい内容

- 問題と潜在的影響の明確な説明
- 再現手順（コマンド、コネクタ、フィクスチャパス、または最小サンプル）
- 影響を受けるコンポーネント（`@zenchi-zenno/core`、CLI、コネクタ、CI ワークフローなど）
- zenchi-zenno のバージョンまたはコミット SHA
- 関連する場合は環境（OS、Node.js バージョン）

## 私たちが約束すること

- **7 日以内**に受領確認（初期フェーズではベストエフォート）
- **30 日以内**にステータス更新
- 修正または緩和策の後に開示を調整
- 報告者が同意した場合、リリースノートでクレジット

## スコープ

対象:

- 配布パッケージにおけるリモートコード実行、認証バイパス、権限昇格
- CLI / コネクタにおけるパストラバーサル、安全でないデシリアライゼーション、コマンドインジェクション
- ログ、イベント、コミット成果物へのシークレットや認証情報の書き込み
- リリースを危険にさらしうる CI/CD ワークフローの弱点

対象外（通常の Issue を使用してください）:

- ソーシャルエンジニアリング
- プロダクト欠陥なしに単一ユーザーのローカル `.zz/` ディレクトリに対する DoS
- zenchi-zenno が誤用していない限り、第三者サービス（GitHub、Google、Slack など）の脆弱性
- ロック解除されたマシンへの物理アクセスが必要な問題

## セーフハーバー

善意のセキュリティ調査を支持します。所有していないデータにアクセスしないでください。他ユーザーのサービスを劣化させないでください。自分のローカルワークスペースと合成フィクスチャでのテストを推奨します。

## 個人データ

zenchi-zenno は個人の知識を取り込みます。報告時は:

- **合成フィクスチャ**または編集済みサンプルを使用する
- 実際の Gmail エクスポート、Slack ダンプ、その他の PII を**添付しない**
- プライベートな内容を共有せずにデータフローの問題を説明する

あわせて [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) と [CONTRIBUTING.md](CONTRIBUTING.md) を参照してください。
