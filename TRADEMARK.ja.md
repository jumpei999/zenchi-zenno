# 商標ポリシー

**zenchi-zenno** の名前、ロゴ、および関連マークの使用に関するポリシー。

**関連:** [GOVERNANCE.md](GOVERNANCE.ja.md) · [docs/commercial-boundary.md](docs/commercial-boundary.ja.md)

---

<a id="ownership"></a>

## 所有権

**zenchi-zenno** の名前、ワードマーク、およびプロジェクトのロゴは、zenchi-zenno プロジェクト / メンテナー (「Project」) の商標です。

ソフトウェアは [LICENSE](LICENSE) (フェーズ 0 ～ 1 の時点では MIT) に基づいてライセンスされています。 **商標権はソフトウェア ライセンスによって付与されません。** コードを使用しても、このポリシーを超えて Project マークを使用する許可は付与されません。

---

<a id="allowed-without-permission"></a>

## 許可なく許可されます

してもいいです：

1. **事実に基づいてプロジェクトを名前で参照します** (「zenchi-zenno との互換性」、「zenchi-zenno のフォーク」)。
2. 公式リポジトリとドキュメントへの**リンク**。
3. zenchi-zenno との統合または拡張に関するドキュメントではこの名前を使用してください**。
4. **互換性を正確に示す** (「zenchi-zenno Connector SPI を実装する」)。

---

<a id="requires-permission"></a>

## 許可が必要です

以下のことを行うには、Project メンテナーからの書面による許可が必要です。

1. **zenchi-zenno** (または紛らわしい類似の名前) をフォーク、ホストされたサービス、または商用製品の **製品名** として使用します。
2. Project **ロゴ**をマーケティング、アプリ ストア、または有料広告で使用します。
3. **公式の承認**、パートナーシップ、または「公式クラウド」ステータスを暗示します。
4. あなたが Project であることを示唆するドメイン、組織、またはソーシャル ハンドルを登録します (例: `zenchi-zenno-official`、`zenchi-zenno.cloud` は公式ステータスを暗示します)。

---

<a id="forks"></a>

## フォーク

ソフトウェアライセンスの下ではフォークは歓迎されます。フォークは**しなければなりません**:

- 競合製品またはホスト型サービスとして配布されている場合は、product/display の名前を変更します。
- Project ロゴは使用しないでください
- 公式の zenchi-zenno ディストリビューションであるとは主張しません

推奨される名前変更: アップストリームとして提示せずに、所属を明確にします (例: README の `acme-knowledge-os (zenchi-zenno fork)`)。

---

<a id="commercial-and-cloud-offerings"></a>

## 商用およびクラウド製品

公式管理製品には、出荷時に個別の名前が付けられます (例: **zenchi-zenno Cloud** など、Project 管理下のブランド名)。 OSS コアを使用するサードパーティのホスト型サービスは、次のことを行う必要があります。

- 紛らわしい類似マークを許可なく使用しないでください
- OSS と SaaS アドオンの説明については、[docs/commercial-boundary.md](docs/commercial-boundary.ja.md) に準拠してください。

<a id="reporting-misuse"></a>

## 不正使用の報告

商標の不正使用の疑いがある場合は、`[Trademark]` というタイトルの GitHub の問題を介して報告するか、リポジトリの所有権設定にリストされているメンテナーに連絡してください。

## アップデート

このポリシーは、Project の成長に応じて更新される可能性があります。重要な変更は、該当する場合、ディスカッションおよび `CHANGELOG` / リリース ノートを通じて発表されます。
